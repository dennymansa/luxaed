# Moving24 - шаблон лид-ген-сайта (референс-кит)

Полный набор для клонирования сайта Moving24 под любой локальный бизнес (заборы, клининг, ремонт и т.д.): файлы сайта + исчерпывающая документация + правила + методология запуска.

## Что где

| Файл / папка | Что это |
|---|---|
| **`site/`** | Чистые файлы сайта Moving24 (без dev-мусора, без `config.php`). Это основа для клона. |
| `site/config.example.php` | Шаблон секретов (Gmail app-password + получатель). Скопировать в `config.php` и заполнить. НЕ коммитить `config.php`. |
| `validate.py`, `build_zip.py` | Тулинг: проверка инвариантов + сборка zip для деплоя. |
| **`GUIDE.md`** | Мастер-гайд (обновлён, ~226 KB, 10 разделов): (1) обзор/архитектура/сборка/деплой, (2) **веб-форма заявки**, (3) **письмо-заявка `lead.php` целиком** (дата DD.MM.YYYY + день недели, кликабельные тел/почта, жёлтый блок сообщения, кнопки Google Maps + Waze, встроенная карта маршрута + расстояние/время, Google Street View адресов), (4) **интеграция Google Maps** (Directions/Static Maps/Street View + подводные камни с ключом), (5) **трекинг/Google Ads** (GTM, Consent Mode, конверсии формы + тел/почта), (6) как устроены **переводы**, (7) редактирование под клиента + темизация, (8) **инвентарь: все страницы и все картинки** (где какая фотка), (9) анатомия страниц/структура/SEO, (10) миграция со старого сайта + производительность. ## Порядок работы над новым сайтом

1. **`LESSONS-FROM-BUILD.md`** - усвой правила (тире, страховка, не трогать GTM, не выдумывать данные и т.д.).
2. **`NEW-CLIENT-PLAYBOOK.md`** - сделай анализ ниши/конкурентов, получи структуру + оценку отклонения.
3. **`GUIDE.md`** - по нему клонируй `site/`: меняй контент, реквизиты, картинки, перекрась, подставь свой GTM/config.
4. **`playbooks/LANG-*.md`** - пиши тексты нативно на нужных языках.
5. `validate.py` -> `build_zip.py` -> деплой (wipe веб-рут -> распаковать -> положить `config.php`).

## Ключевые правила (шпаргалка)

- Тире `-` только дефис. Без страховки. Не выдумывать цифры/отзывы.
- GTM-контейнер и рекламный кабинет НЕ трогать - сайт воспроизводит их события.
- `config.php` и dev-мусор - никогда не в публичный доступ.
- Деплой: не поверх старого сайта; на проде снять `noindex`; старые URL -> 301.
- Проверять на живом инструментально (curl / dataLayer / CSSOM), а не на глаз.
